## Key Conventions

### Server vs Client Components
- Pages are **Server Components** by default вЂ” they fetch data directly from Prisma
- Pages pass data to `*-client.tsx` **Client Components** for interactivity
- API routes are used for mutations (POST, PATCH, DELETE), not for initial page loads

### Route Protection
- `src/middleware.ts` handles auth at the edge вЂ” no page needs to redirect manually except as a secondary check in the layout

### Data Flow Pattern
```
Server Page в†’ fetches from Prisma в†’ passes to Client Component
Client Component в†’ calls /api/* for mutations в†’ router.refresh() to revalidate
```

### Adding a New Feature
1. Add model to `prisma/schema.prisma` в†’ `npm run db:migrate`
2. Add API route in `src/app/api/`
3. Add server page in `src/app/(dashboard)/`
4. Add client component in `src/components/`
